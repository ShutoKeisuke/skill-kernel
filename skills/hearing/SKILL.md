# ヒアリングスキル

## 概要

Web開発の案出しに必要な情報をユーザーからヒアリングする。
ヒアリング完了後、収集した情報を構造化してまとめる。

## フェーズ1：フリー入力

まず以下のメッセージを表示し、ユーザーの自由入力を受け付ける。

```
💡 Web開発の案出しを始めましょう。

どんな開発をしたいか、思っていることを自由に教えてください。
（何もなければ Enter でスキップできます）

>
```

- 入力あり → その内容を起点に深掘り質問を生成する
- 入力なし → 「開発の目的」を起点に質問を開始する

## フェーズ2：選択式ヒアリング（最大10往復）

### 質問の形式

各質問は必ず **2〜4択の選択肢** として提示する。

```
質問 N/最大10：{質問文}

1. {選択肢A}
2. {選択肢B}
3. {選択肢C}
4. わからない / まだ決めていない

>
```

### ヒアリング方針

- 質問はコンテキストに応じて動的に生成する（固定リストではない）
- 十分な情報が集まったと判断した場合は10往復未満で終了して構わない
- 最後の質問終了後に「ヒアリングを完了します。」と表示する

### 収集すべき観点

以下の情報を優先度順に収集する。

| 優先度 | 観点 | 例 |
|--------|------|----|
| 高 | 開発の目的・背景 | 業務効率化、新規サービス立ち上げ |
| 高 | ターゲットユーザー | 一般消費者、社内スタッフ、特定業種 |
| 高 | 主要な解決したい課題 | 手作業が多い、情報が分散している |
| 中 | 技術的な制約・希望 | 既存システムとの連携、特定スタックの指定 |
| 中 | 開発リソース | 人数・期間の見込み |
| 低 | 運用・コスト面の制約 | 月額予算の上限、運用担当者の有無 |

## フェーズ3：ヒアリング結果のまとめ

ヒアリング完了後、以下の構造でまとめを出力する。

```
## ヒアリング結果

- **目的・背景**: {内容}
- **ターゲットユーザー**: {内容}
- **解決したい課題**: {内容}
- **技術的な制約・希望**: {内容 or "特になし"}
- **開発リソース**: {内容 or "未定"}
- **運用・コスト制約**: {内容 or "未定"}
```

このまとめを後続の案生成フェーズに引き渡す。
